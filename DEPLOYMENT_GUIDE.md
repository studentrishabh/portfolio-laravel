## 🖥️ PART 1: Local Development Setup

### Step 1: Prerequisites – Install Required Software

Install these on your machine first:

| Software | Download Link | Why Needed |
|----------|--------------|------------|
| PHP 8.1+ | https://www.php.net/downloads | Runs Laravel |
| Composer | https://getcomposer.org | PHP package manager |
| MySQL 8.0 | https://dev.mysql.com/downloads | Database |
| Node.js 18+ | https://nodejs.org | Runs Vite + Vue build |
| Git | https://git-scm.com | Version control |

**Verify installations:**
```bash
php --version        # Should show PHP 8.1+
composer --version   # Should show Composer 2.x
mysql --version      # Should show MySQL 8.x
node --version       # Should show v18+
npm --version        # Should show 9+
git --version        # Should show git 2.x
```

---

### Step 2: Install Laravel 10 Project

**Option A: Clone from GitHub (after you push)**
```bash
git clone https://github.com/YOUR_USERNAME/portfolio.git
cd portfolio
```

**Option B: Create fresh Laravel 10 project and copy files**
```bash
composer create-project laravel/laravel portfolio "^10.0"
cd portfolio
# Then copy all the files from this project into the Laravel folder
```

**Install PHP dependencies:**
```bash
composer install
```

This downloads all packages listed in `composer.json` into the `vendor/` folder.

---

### Step 3: Configure the .env File

```bash
# Copy the example env file
cp .env.example .env

# Generate a unique app key (required by Laravel for encryption)
php artisan key:generate
```

Now open `.env` in any text editor and update:
```env
APP_NAME="Rishabh Portfolio"
APP_ENV=local
APP_DEBUG=true
APP_URL=http://localhost:8000

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=portfolio_db
DB_USERNAME=root
DB_PASSWORD=your_mysql_password
```

**Why `.env` exists:**
Laravel reads configuration from this file so you can have different settings locally vs on the live server, without changing code.

---

### Step 4: Setup MySQL Database

**Open MySQL in terminal:**
```bash
mysql -u root -p
# Enter your MySQL password when prompted
```

**Run these SQL commands:**
```sql
-- Create a dedicated database for the portfolio
CREATE DATABASE portfolio_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- Verify it was created
SHOW DATABASES;

-- Exit MySQL
EXIT;
```

**Or use phpMyAdmin / MySQL Workbench:**
1. Open phpMyAdmin at `http://localhost/phpmyadmin`
2. Click "New" on the left sidebar
3. Database name: `portfolio_db`
4. Collation: `utf8mb4_unicode_ci`
5. Click "Create"

---

### Step 5: Run Migrations

```bash
php artisan migrate
```

This command reads all files in `database/migrations/` and creates the `contacts` table in MySQL.

**Verify the table was created:**
```bash
php artisan tinker
>>> \App\Models\Contact::count()
# Should return 0 (table exists, no records yet)
>>> exit
```

---

### Step 6: Add Your Resume PDF

Place your resume PDF in:
```
public/resume/rishabh-chaurasiya-resume.pdf
```

Create the directory if needed:
```bash
mkdir -p public/resume
# Then copy your PDF file there
```

The download button in the hero and navbar will link to this file automatically.

---

### Step 7: Install Vue.js / Frontend Dependencies

```bash
npm install
```

This downloads Vue 3, Vite, and Axios into `node_modules/` based on `package.json`.

---

### Step 8: Run Vue Frontend (Development Mode)

**Option A: Development with hot reload (recommended while coding)**

Open **two terminal windows**:

**Terminal 1 – Laravel backend:**
```bash
php artisan serve
# Runs at http://127.0.0.1:8000
```

**Terminal 2 – Vue dev server:**
```bash
npm run dev
# Vite starts at http://localhost:5173
# Hot Module Replacement (HMR) active — changes appear instantly
```

Visit `http://localhost:5173` in your browser to see the portfolio.

---

### Step 9: Build Production Files

When you're done developing and ready to deploy:

```bash
npm run build
```

This compiles all Vue components into `public/js/app.js` (minified, optimized).

After building, you can test the production build using only Laravel:
```bash
php artisan serve
# Visit http://127.0.0.1:8000
```

## 🌐 PART 3: Free Deployment Options

### Option A: Deploy on Railway (Recommended – Free, Easy)

**Railway supports Laravel with MySQL — best option.**

1. Go to [https://railway.app](https://railway.app)
2. Sign in with GitHub
3. Click **"New Project"** → **"Deploy from GitHub repo"**
4. Select your `portfolio` repository
5. Railway auto-detects PHP/Laravel

**Add MySQL:**
- Click **"+ New"** → **"Database"** → **"MySQL"**
- Railway provisions a free MySQL database

**Set Environment Variables:**
Click your app → **"Variables"** tab → Add:
```
APP_NAME=Rishabh Portfolio
APP_ENV=production
APP_KEY=base64:YOUR_KEY_HERE   ← Run: php artisan key:generate --show
APP_DEBUG=false
APP_URL=https://YOUR-APP.railway.app

DB_CONNECTION=mysql
DB_HOST=${{MySQL.MYSQL_HOST}}
DB_PORT=${{MySQL.MYSQL_PORT}}
DB_DATABASE=${{MySQL.MYSQL_DATABASE}}
DB_USERNAME=${{MySQL.MYSQL_USER}}
DB_PASSWORD=${{MySQL.MYSQL_PASSWORD}}
```

**Add build commands:**
Create a `Procfile` in project root:
```
web: php artisan serve --host=0.0.0.0 --port=$PORT
```

Create `railway.json`:
```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "php artisan migrate --force && php artisan serve --host=0.0.0.0 --port=$PORT"
  }
}
```

---

### Option B: Deploy on Render (Free)

1. Go to [https://render.com](https://render.com)
2. Sign in with GitHub
3. Click **"New"** → **"Web Service"**
4. Connect your GitHub repo
5. Settings:
   - **Runtime:** PHP
   - **Build Command:** `composer install --no-dev && npm install && npm run build`
   - **Start Command:** `php artisan serve --host=0.0.0.0 --port=$PORT`
6. Add environment variables (same as Railway above)
7. Add MySQL: **"New"** → **"PostgreSQL"** (or use external MySQL like PlanetScale)

---

### Option C: Deploy on Shared Hosting (cPanel – InfinityFree, etc.)

For free shared hosting like [InfinityFree](https://infinityfree.net):

**Step 1: Build locally first**
```bash
npm run build
composer install --no-dev --optimize-autoloader
php artisan config:cache
php artisan route:cache
```

**Step 2: Upload files**
- Upload entire project to `public_html/portfolio/` via FTP (use FileZilla)
- Move contents of `public/` to `public_html/` (or create a subdomain)

**Step 3: Configure document root**
The web server must point to Laravel's `public/` folder, not the project root.

In cPanel → **"Subdomains"** or **"Addon Domains"**:
- Document Root: `/home/username/portfolio/public`

**Step 4: Create .env on server**
Via cPanel File Manager, create `.env` with production values.

**Step 5: Run migrations via SSH**
```bash
cd /home/username/portfolio
php artisan migrate --force
```

## ⚙️ PART 4: Configure Environment Variables on Live Server

**Critical production .env settings:**

```env
# Always set these for production
APP_ENV=production
APP_DEBUG=false          ← IMPORTANT: false hides error details from visitors
APP_KEY=base64:...       ← Generate with: php artisan key:generate --show

# Database (use your live server credentials)
DB_HOST=your_live_db_host
DB_DATABASE=your_db_name
DB_USERNAME=your_db_user
DB_PASSWORD=your_strong_password

# Public URL of your portfolio
APP_URL=https://yourdomain.com
```

**After updating .env on server, run:**
```bash
php artisan config:clear
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

## 🔧 Common Issues & Fixes

| Problem | Fix |
|---------|-----|
| `php artisan` not working | Ensure PHP is in PATH: `export PATH=$PATH:/usr/bin/php` |
| DB connection refused | Check MySQL is running: `sudo service mysql start` |
| 500 error on live server | Set `APP_DEBUG=true` temporarily to see error, then set back to `false` |
| Vue not loading | Run `npm run build` and ensure `public/js/app.js` exists |
| Contact form CORS error | Set `APP_URL` correctly in `.env` on live server |
| Missing app key | Run `php artisan key:generate` |
| Migrations fail | Ensure DB credentials in `.env` are correct and DB exists |
